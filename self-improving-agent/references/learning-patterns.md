# Learning Patterns

Common patterns for learning and self-improvement.

## Pattern 1: Feedback Loop

```
Action → Result → Feedback → Adjustment → Action
```

**Application:**
- Try an approach
- Observe the result
- Get explicit or implicit feedback
- Adjust behavior
- Try again with improvements

**Example:**
1. Provide verbose response
2. User gives brief reply
3. Infer user prefers conciseness
4. Adjust to be more concise
5. Monitor future interactions

## Pattern 2: Error-Driven Learning

```
Error → Analysis → Pattern Detection → Prevention → Validation
```

**Application:**
- Make a mistake
- Analyze root cause
- Identify if it's a pattern
- Implement prevention
- Verify it works

**Example:**
1. Fail to check file existence before reading
2. Analyze: Assumed file exists
3. Pattern: Often forget existence checks
4. Prevention: Add check to workflow
5. Validate: No more file errors

## Pattern 3: Success Replication

```
Success → Deconstruction → Generalization → Application → Validation
```

**Application:**
- Have a success
- Break down what worked
- Generalize the approach
- Apply to similar situations
- Confirm it works again

**Example:**
1. Successfully debug complex issue
2. Deconstruct: Systematic elimination approach
3. Generalize: Divide-and-conquer debugging
4. Apply to next debugging task
5. Confirm success rate improves

## Pattern 4: Preference Accumulation

```
Observation → Hypothesis → Test → Confirmation → Documentation
```

**Application:**
- Notice user behavior
- Form hypothesis about preference
- Test with adjusted behavior
- Confirm or refine hypothesis
- Document confirmed preference

**Example:**
1. Notice user edits out formal greetings
2. Hypothesis: User prefers direct style
3. Test: Skip formalities next time
4. Confirm: User responds positively
5. Document: "User prefers direct communication"

## Pattern 5: Skill Stacking

```
Skill A + Skill B → Combined Capability → New Applications
```

**Application:**
- Master individual skills
- Combine skills in new ways
- Discover emergent capabilities
- Apply to novel situations

**Example:**
1. Good at file reading
2. Good at web searching
3. Combine: Research assistant capability
4. Apply: "Find and summarize articles"

## Pattern 6: Context Transfer

```
Domain A Learning → Abstraction → Domain B Application
```

**Application:**
- Learn something in one context
- Abstract the principle
- Apply to different domain
- Validate transfer works

**Example:**
1. Learn user prefers bullet points in summaries
2. Abstract: Structured formats preferred
3. Apply: Use tables for data in other tasks
4. Validate: User appreciates structure

## Pattern 7: Progressive Refinement

```
Initial Attempt → Partial Success → Iteration → Improvement → Mastery
```

**Application:**
- Make initial attempt
- Get partial results
- Iterate based on feedback
- Gradually improve
- Achieve mastery

**Example:**
1. First code review: catches basic issues
2. Misses some edge cases
3. Add edge case checklist
4. Improvement: catch more issues
5. Mastery: comprehensive reviews

## Anti-Patterns to Avoid

### Over-Generalization
- **Problem:** Assuming one learning applies everywhere
- **Solution:** Validate generalizations before applying broadly

### Confirmation Bias
- **Problem:** Only noticing evidence that supports existing beliefs
- **Solution:** Actively look for disconfirming evidence

### Premature Optimization
- **Problem:** Optimizing before understanding the real bottleneck
- **Solution:** Measure first, optimize based on data

### Memory Hoarding
- **Problem:** Keeping everything "just in case"
- **Solution:** Regularly prune outdated or irrelevant memories
