# Reflection Prompts

Structured prompts for different types of reflection scenarios.

## Post-Task Reflection

Use after completing any significant task:

```
Task: [Brief description of what was done]

Reflection Questions:
1. What was the goal and was it achieved?
2. What approach was taken?
3. What worked well?
4. What could have been done better?
5. Were there any unexpected challenges?
6. What would I do differently next time?
7. Any user feedback or reactions?

Action Items:
- [ ] Update MEMORY.md with lessons learned
- [ ] Document any new patterns or approaches
- [ ] Note any skill improvements needed
```

## Mistake Analysis

Use when something goes wrong:

```
Mistake: [What went wrong]

Analysis:
1. What was the root cause?
2. Was this a one-time error or a pattern?
3. What signals could have predicted this?
4. How was it detected and corrected?

Prevention:
- [ ] Add check/procedure to prevent recurrence
- [ ] Update relevant skill documentation
- [ ] Share learning in MEMORY.md

Recovery:
- How was the issue resolved?
- What was the user impact?
```

## Success Pattern Recognition

Use when something goes particularly well:

```
Success: [What went well]

Pattern Analysis:
1. What made this successful?
2. Can this approach be generalized?
3. What conditions enabled success?
4. How can this be replicated?

Documentation:
- [ ] Document the successful approach
- [ ] Consider adding to skill templates
- [ ] Note for future similar tasks
```

## User Preference Discovery

Use when learning about user preferences:

```
Observation: [What was noticed about user preferences]

Context:
- When did this occur?
- What was the situation?
- How was the preference expressed?

Implications:
- How should behavior change?
- What should be remembered?
- Are there related preferences?

Action:
- [ ] Update USER.md or MEMORY.md
- [ ] Adjust future interactions accordingly
```

## Periodic Review (Weekly/Monthly)

Use during scheduled reflection time:

```
Review Period: [Date range]

Interaction Summary:
- Total interactions: [count]
- Tasks completed: [count]
- User feedback received: [summary]

Patterns Identified:
1. [Pattern 1 with examples]
2. [Pattern 2 with examples]

Key Learnings:
1. [Learning 1]
2. [Learning 2]

Memory Maintenance:
- [ ] Consolidate daily notes into MEMORY.md
- [ ] Remove outdated information
- [ ] Update performance metrics
- [ ] Archive old daily notes
```
